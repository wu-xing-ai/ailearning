## API工具层

文件路径：`frontend/src/utils/api.js`

统一的API调用封装，自动处理JWT认证头和响应解析。

### 通用方法

#### `api.get(url)`

通用GET请求，自动附加认证头。

| 参数 | 类型 | 说明 |
|------|------|------|
| url | string | 请求路径 |
| **返回值** | Promise\<object\> | 解析后的JSON数据 |

#### `api.post(url, data, options)`

通用POST请求，自动附加认证头。

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| url | string | 是 | 请求路径 |
| data | object | 否 | 请求体 |
| options | object | 否 | 额外配置（如 `responseType: 'stream'`） |
| **返回值** | Promise\<object\> | 解析后的JSON数据 |

---

### 对话接口

#### `api.postChat(prompt, stream, modelConfig)`

与AI进行对话。

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| prompt | string | 是 | 用户输入 |
| stream | boolean | 否 | 是否流式，默认true |
| modelConfig | object | 否 | 模型配置 `{type, name, apiKey, apiUrl}` |
| **返回值** | Promise\<Response\> | 原始Response对象（用于流式读取） |

**调用示例:**
```javascript
// 流式调用
const response = await api.postChat('解释量子力学', true, {
  type: 'custom',
  name: 'glm-5',
  apiKey: 'sk-xxx',
  apiUrl: 'https://api.example.com/v1'
})
const reader = response.body.getReader()
// 逐块读取SSE数据...

// 非流式调用
const response = await api.postChat('你好', false)
```

**对应后端:** `POST /api/ai/chat`

## 认证模块

文件路径：`frontend/src/stores/auth.js`

基于Pinia的状态管理，提供用户认证状态和方法。

### 状态

| 状态 | 类型 | 说明 |
|------|------|------|
| user | object \| null | 当前用户信息 |
| token | string \| null | JWT Token |

### 计算属性

| 属性 | 类型 | 说明 |
|------|------|------|
| isAuthenticated | boolean | 是否已登录 |
| isAdmin | boolean | 是否为管理员 |
| isTeacher | boolean | 是否为教师或管理员 |

### 方法

#### `login(username, password)`

用户登录，成功后存储token和用户信息到localStorage。

| 参数 | 类型 | 说明 |
|------|------|------|
| username | string | 用户名 |
| password | string | 密码 |

**对应后端:** `POST /api/auth/login`

**调用示例:**
```javascript
const authStore = useAuthStore()
await authStore.login('student1', 'password')
console.log(authStore.user.username) // 'student1'
```

---

#### `register(data)`

用户注册。

| 参数 | 类型 | 说明 |
|------|------|------|
| data | object | `{username, password, display_name, email, role}` |

**对应后端:** `POST /api/auth/register`

---

#### `logout()`

退出登录，清除token和用户信息。

---

#### `initFromStorage()`

从localStorage恢复登录状态（应用启动时调用）。

- Token 存储在 `localStorage('auth_token')`

## 请求规范

### 认证方式

所有需要认证的请求在请求头中携带JWT Token：

```javascript
headers: { 'Authorization': 'Bearer <token>' }
```

Token存储在 `localStorage.getItem('auth_token')`，由 `api.js` 的 `getAuthHeaders()` 自动附加。

### 代理配置

开发环境通过Vite代理转发API请求：

```javascript
// vite.config.js
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8000',
      changeOrigin: true
    }
  }
}
```

生产环境通过nginx反向代理 `/api` 到后端服务。

### 响应处理

- JSON响应自动解析
- 非JSON响应返回 `{detail: text}`
- HTTP错误抛出 `Error`，message为 `detail` 字段内容
- 流式响返回原始 `Response` 对象，由调用方通过 `ReadableStream` 读取SSE数据

### 错误处理模式

```javascript
try {
  const result = await api.getDocuments()
  // 处理成功
} catch (error) {
  // error.message 包含后端返回的 detail 信息
  ElMessage.error(error.message)
}
```
