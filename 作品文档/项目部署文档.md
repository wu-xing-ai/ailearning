## 方式一：Docker部署（推荐）

### 1. 获取项目

```bash
git clone <项目地址>
cd ailearning
```

### 2. 配置环境变量（可选）

Docker部署已内置默认配置，开箱即用。如需自定义，在项目根目录创建 `.env` 文件：

```bash
# MySQL密码（默认 ailearning123）
MYSQL_ROOT_PASSWORD=your_password

# Ollama地址（如果使用本地Ollama）
OLLAMA_BASE_URL=http://host.docker.internal:11434

# ModelScope API Key（内置默认Key，可替换为自己的）
MODELSCOPE_API_KEY=your_key
```

### 3. 一键启动

```bash
# 使用部署脚本
chmod +x deploy.sh
./deploy.sh start

# 或直接使用docker compose
docker compose up -d --build
```

### 4. 验证部署

```bash
# 查看服务状态
./deploy.sh status

# 查看日志
./deploy.sh logs
```

- 前端访问：`http://localhost`
- 后端API：`http://localhost:8000`
- API文档：`http://localhost:8000/docs`

### 5. 管理命令

```bash
./deploy.sh start      # 启动服务
./deploy.sh stop       # 停止服务
./deploy.sh restart    # 重启服务
./deploy.sh logs       # 查看日志
./deploy.sh status     # 查看状态
```

### Docker容器说明

| 容器 | 端口 | 说明 |
|------|------|------|
| ailearning-mysql | 3306 | MySQL 8.0 数据库 |
| ailearning-backend | 8000 | FastAPI 后端服务 |
| ailearning-frontend | 80 | Nginx 前端服务 |

### 数据持久化

Docker使用Volume持久化数据：

| Volume | 挂载点 | 说明 |
|--------|--------|------|
| mysql_data | MySQL数据目录 | 数据库文件 |
| pip_cache | /root/.cache/pip | Python包缓存 |
| npm_cache | /root/.npm | npm包缓存 |
| ./backend/knowledge_base | /app/knowledge_base | 上传的文档文件 |
| ./backend/chroma_data | /app/chroma_data | 向量数据 |

## 方式二：本地开发部署

### 1. 准备MySQL数据库

```bash
# 安装MySQL 8.0后，创建数据库
mysql -u root -p
CREATE DATABASE ailearning CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2. 配置后端

```bash
cd backend

# 创建环境变量文件
cp .env.example .env
```

编辑 `backend/.env`：

```bash
# 数据库连接（必须修改）
DATABASE_URL=mysql+pymysql://root:your_password@localhost:3306/ailearning?charset=utf8mb4

# JWT密钥（必须修改）
JWT_SECRET_KEY=your-random-secret-string

# AI API Keys（按需填写）
SILICONFLOW_API_KEY=your_key
MODELSCOPE_API_KEY=your_key

# 向量嵌入配置（按需填写）
EMBEDDING_API_KEY=your_key
EMBEDDING_API_URL=https://api.siliconflow.cn/v1
EMBEDDING_MODEL=BAAI/bge-large-zh-v1.5
EMBEDDING_DIMENSIONS=1024
```

### 3. 安装后端依赖并启动

```bash
cd backend

# 创建虚拟环境（推荐）
python -m venv venv
source venv/bin/activate    # Linux/Mac
# venv\Scripts\activate     # Windows

# 安装依赖
pip install -r requirements.txt

# 启动服务
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

后端启动后访问 `http://localhost:8000/docs` 查看API文档。

### 4. 安装前端依赖并启动

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端访问 `http://localhost:5173`，开发环境会自动代理 `/api` 请求到 `localhost:8000`。

### 5. 构建前端生产版本

```bash
cd frontend
npm run build
```

构建产物在 `frontend/dist/` 目录，可部署到任意Web服务器。

## 配置说明

### 配置文件结构

```
backend/
├── .env                  # 敏感配置（API Key、数据库密码、JWT密钥）
├── .env.example          # 配置模板
├── config.json           # 非敏感配置（API地址、模型名称等）
├── ollama_config.json    # Ollama服务配置
└── custom_models.json    # 自定义模型配置
```

### 配置优先级（从高到低）

1. `.env` 环境变量
2. 运行时参数（API调用传入）
3. `config.json` 默认值

### 核心环境变量

| 变量 | 说明 | 示例 |
|------|------|------|
| DATABASE_URL | 数据库连接 | `mysql+pymysql://root:pwd@localhost:3306/ailearning?charset=utf8mb4` |
| JWT_SECRET_KEY | JWT签名密钥 | 随机字符串 |
| SILICONFLOW_API_KEY | SiliconFlow API Key | `sk-xxx` |
| MODELSCOPE_API_KEY | ModelScope API Key | `ms-xxx` |
| OPENAI_API_KEY | OpenAI API Key | `sk-xxx` |
| ANTHROPIC_API_KEY | Anthropic API Key | `sk-ant-xxx` |
| ZHIPU_API_KEY | 智谱AI API Key | `xxx.xxx` |
| QWEN_API_KEY | 通义千问 API Key | `sk-xxx` |
| EMBEDDING_API_KEY | 向量嵌入API Key | `sk-xxx` |
| VL_API_KEY | 视觉模型API Key | `sk-xxx` |

### Nginx配置说明（生产环境）

```nginx
# Vue Router history模式
location / {
    try_files $uri $uri/ /index.html;
}

# API反向代理
location /api/ {
    proxy_pass http://backend:8000/api/;
    # SSE流式响应支持
    proxy_buffering off;
    proxy_cache off;
    proxy_http_version 1.1;
}

# 上传文件大小限制
client_max_body_size 60m;

# 静态资源缓存
location /assets/ {
    expires 30d;
}
```

## 服务架构

```
                    ┌─────────────┐
                    │   浏览器     │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │   Nginx     │  端口 80
                    │  (前端静态)  │
                    └──────┬──────┘
                           │ /api/*
                    ┌──────▼──────┐
                    │  FastAPI    │  端口 8000
                    │  (后端API)  │
                    └──┬───┬───┬─┘
                       │   │   │
              ┌────────┘   │   └────────┐
              ▼            ▼            ▼
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │  MySQL   │ │ AI API   │ │ 文件存储  │
        │  (数据库) │ │ (多厂商) │ │ (本地磁盘)│
        └──────────┘ └──────────┘ └──────────┘
```

### 支持的AI厂商

| 厂商 | 需要API Key | 默认模型 |
|------|------------|---------|
| Ollama（本地） | 否 | llama3.2, qwen2 |
| OpenAI | 是 | gpt-4, gpt-3.5-turbo |
| Anthropic | 是 | claude-3-opus, claude-3-sonnet |
| 智谱AI (GLM) | 是 | glm-4, glm-4-air |
| 阿里通义千问 | 是 | qwen-turbo, qwen-plus |

## 常见问题

### 1. 数据库连接失败

检查 `.env` 中的 `DATABASE_URL` 是否正确，MySQL服务是否启动。

```bash
# Docker环境检查MySQL状态
docker compose ps mysql

# 查看MySQL日志
docker compose logs mysql
```

### 2. 前端无法访问后端API

- Docker环境：检查nginx配置中 `proxy_pass` 是否指向 `backend:8000`
- 本地开发：检查 `vite.config.js` 中代理配置是否指向 `localhost:8000`

### 3. 上传文件失败

- 检查文件大小是否超过50MB（后端限制）或60MB（nginx限制）
- 检查 `backend/knowledge_base` 目录是否有写权限

### 4. AI服务不可用

- Ollama：确认Ollama服务已启动，Docker环境使用 `host.docker.internal` 访问宿主机
- 云端API：检查对应的API Key是否正确配置

### 5. 端口被占用

修改 `docker-compose.yml` 中的端口映射：

```yaml
ports:
  - "8080:80"      # 前端改为8080
  - "8001:8000"    # 后端改为8001
```

### 6. 数据库重置

```bash
# 停止服务并删除数据库Volume
docker compose down -v

# 重新启动（会自动初始化数据库）
docker compose up -d --build
```

> 注意：删除Volume会丢失所有数据，请先备份。
